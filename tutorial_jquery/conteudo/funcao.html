Basicamente, por que utilizar a jQuery?

O problema

A linguagem Javascript permite que os programadores modifiquem o estilo, conteúdo e comportamento dos elementos de uma página Web. Mais especificamente, programadores podem "anexar" manipuladores de eventos diretamente no HTML, como abaixo:

<div id="conteudo" onclick="alert('Clicou!')" >...</div>

Mesmo que o exemplo acima represente uma maneira possível e correta (de acordo com padrões) de se manipular eventos em Javascript, existem diversos problemas que podem ocorrer com esse tipo de abordagem, entre esses:

  • Resulta em uma estrutura monolítica, onde não se separa lógica de programação (Javascript) de estrutura de apresentação (HTML).
  • Cria uma tendência de repetição desnecessária e de dispersão do código dentro do HTML.
O que acaba dificultando tanto a manutenção do código quanto a diversos outros aspectos no desenvolvimento (como a divisão de tarefas entre programadores e designers, por exemplo).

jQuery como uma solução

Com a jQuery, temos uma forma eficiente (e fácil) de separar o comportamento da aplicação de sua estrutura de apresentação. A biblioteca disponibiliza tais funcionalidades através de seletores (padrão CSS3) e de manipuladores de evento internos. Além de outras diversas funcionalidades.
